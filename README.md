# digital-brain

Gitignored Obsidian vault as Claude's "second brain" for any project. Cuts token usage 8-15× on orientation questions: Claude reads concept summaries + code graph instead of grepping source.

**Status: v0 MVP (2026-05-23). Standalone repo.**

What works in v0:
- **SessionStart auto-behavior** — open `claude` in a brain-enabled project; if the vault exists, the INDEX loads automatically (~2k tok); if not, Claude prompts you to build it.
- `/refresh-digital-brain` — full rebuild (AST extract + filter + cluster + Obsidian export + Claude-written concept notes + Obsidian auto-register + INDEX read into context).
- `/load-digital-brain` — manual fallback to load the INDEX mid-session.
- Auto-refresh extracted layer on `git commit` (post-commit hook, no LLM, ~3 sec).
- Stop hook warns Claude when vault is stale (≥5 commits behind or ≥7 days old).
- Bidirectional node references: Claude cites with `[[X]] ([source](src/x.py#L42), [vault](obsidian://...))`;  user types `[[X]]` and the skill resolves it to the vault file.
- Per-project config: `.digital-brain-config.yaml`.
- `digital-brain remove` (per-project) and `digital-brain uninstall` (global) for clean removal.

Deferred to v1: `/init-brain` auto-bootstrap, incremental refresh, business notes, focused-load `/load-digital-brain <keyword>`, LLM-driven community labels, `.last-refresh-diff.md` code-review flow.

---

## What it does

The vault has four layers, each with a distinct purpose:

- **Extracted** (`type: code`) — one note per class/function, auto-generated by graphify AST; wiped and rebuilt on every refresh. Do not edit.
- **Community** (`_COMMUNITY_N.md`) — auto-labeled clusters of related nodes; identifies major subsystems. Do not edit.
- **Concept** (`layer: concept`) — hand-written or Claude-written 150-300 word summaries of design decisions; survive every refresh. Edit freely.
- **INDEX** (`_INDEX.md`) — machine-generated map of all nodes + communities; the only file Claude auto-loads at session start. Do not edit.

## Install (one-time global)

```bash
git clone <repo> ~/Projects/digital-brain
cd ~/Projects/digital-brain/helpers
python3 -m venv .venv
.venv/bin/pip install -e ".[dev]"
.venv/bin/pytest -v                  # expect: all passed

cd ..
bash hooks/install.sh
```

`install.sh` creates skill symlinks under `~/.claude/skills/`, patches `~/.claude/settings.json` with SessionStart + Stop hooks, symlinks `digital-brain` into `~/.local/bin/`, and writes `~/.digital-brain/install.json`. It is idempotent — safe to re-run after updates.

If `~/.local/bin` is not on your `$PATH`, the installer prints the exact line to add to `~/.zshrc` or `~/.bashrc`.

## Per-project setup

```bash
cd <host-repo>

# 1. Write config (commit this file)
cat > .digital-brain-config.yaml <<'EOF'
source_paths:
  - src/
vault_dir: digital-brain/
EOF

# 2. Gitignore the vault
echo "digital-brain/" >> .gitignore

# 3. Symlink the post-commit hook
INSTALL_ROOT="$(python3 -c "import json,os; print(json.load(open(os.path.expanduser('~/.digital-brain/install.json')))['install_root'])")"
ln -sf "$INSTALL_ROOT/hooks/post-commit" .git/hooks/post-commit

# 4. Open Claude — it prompts you to build
claude
```

On first open, the SessionStart hook detects the missing vault and injects:

> "No digital-brain vault detected. Run `/refresh-digital-brain` to build it (~1-3 min)."

Run `/refresh-digital-brain`. On subsequent sessions the INDEX loads automatically.

**Multi-path example:**

```yaml
source_paths:
  - packages/api/src/
  - packages/worker/src/
vault_dir: digital-brain/
```

## Daily workflow

| When | Action | Cost |
|------|--------|------|
| New Claude session (vault exists) | Nothing — INDEX auto-loaded by SessionStart hook | ~2k tok |
| New Claude session (vault missing) | Claude prompts → run `/refresh-digital-brain` | ~1-3 min (Claude writes concept notes) |
| Reference a node mid-session | Type `[[NodeName]]` — skill resolves to vault file | ~1 file read |
| `git commit` touching source_paths | Nothing — post-commit hook fires automatically | ~3 sec, no LLM |
| Big refactor complete, concept notes stale | `/refresh-digital-brain` interactively | ~1-3 min |
| Stop hook warned "vault behind" | `/refresh-digital-brain` | same |
| Mid-session INDEX reload needed | `/load-digital-brain` | ~2k tok |

### What fires automatically

**Post-commit hook** (`hooks/post-commit`):
- Runs only if the commit touched `source_paths`. Skips silently otherwise.
- Runs `auto_refresh.py` in background: graphify AST extract → filter (drops `__init__`, private helpers, dunders, docstring nodes >80 chars) → cluster → drop communities <5 members → Obsidian export → INDEX rewrite.
- Concept notes preserved under `<vault>/.history/<timestamp>/` before extracted layer rebuild.
- Updates `_INDEX.md` `last_refresh_commit` to current HEAD.
- Does NOT refresh concept notes (LLM required — use `/refresh-digital-brain` manually).

**SessionStart hook** (`hooks/session_start.sh`):
- Fires at the start of every `claude` session.
- Walks up the directory tree looking for `.digital-brain-config.yaml`.
- No config found → silent exit (non-brain project, zero overhead).
- Config found, vault or `_INDEX.md` missing → prompts to build.
- Config + vault present → injects `_INDEX.md` contents into session context.
- If INDEX exceeds 9,500 chars (Claude Code's context cap), truncates and appends a pointer so Claude can read the full file on demand.

**Stop hook** (`hooks/stale_check.sh`):
- Runs after each Claude turn.
- Stale if: ≥5 commits behind or INDEX older than 7 days.
- When stale: emits JSON with a system message (UI) and `additionalContext` (Claude sees it) suggesting `/refresh-digital-brain`.
- Tunable: `DIGITAL_BRAIN_STALE_COMMITS=N DIGITAL_BRAIN_STALE_DAYS=N` env vars.

## Removal

### Per-project

Run from inside the host repo:

```bash
digital-brain remove
```

Confirms interactively, then:
1. Copies `layer: concept` notes to `~/.digital-brain-graveyard/<project>-<timestamp>/`.
2. Deletes `<vault_dir>/` and `.digital-brain-config.yaml`.
3. Removes `.git/hooks/post-commit` symlink (only if it resolves to our hook).
4. Unregisters the vault from Obsidian.

Use `--yes` / `-y` to skip confirmation (scripting / CI).

### Global

```bash
digital-brain uninstall
```

Removes skill symlinks, our hook entries from `~/.claude/settings.json`, the `~/.local/bin/digital-brain` symlink, and `~/.digital-brain/install.json`. Never touches per-project vaults or configs — run `digital-brain remove` inside each project first.

The source repo at `<install_root>` is left intact. `rm -rf` it manually if you want it gone.

## Editable vs auto-generated notes

| Type | Frontmatter marker | Editable? | Survives refresh? |
|------|--------------------|-----------|-------------------|
| Concept | `layer: concept` | YES — write freely | YES, preserved |
| Extracted (graphify) | `type: "code"`, no `layer` | NO — auto-regenerated | NO — wiped; copy in `.history/` |
| Community | filename `_COMMUNITY_N.md` | NO | NO |
| Index | `_INDEX.md` | NO | NO |
| Canvas | `graph.canvas` | NO | NO |

If you accidentally edit an extracted note and a refresh wipes it, recover from `<vault>/.history/<timestamp>/`.

## Troubleshooting

| Symptom | Fix |
|---------|-----|
| `/refresh-digital-brain` or `/load-digital-brain` not in skill list | Symlinks broken: `ls -la ~/.claude/skills/`; re-run `bash hooks/install.sh` |
| `ModuleNotFoundError: digital_brain_helpers` | `cd <install_root>/helpers && .venv/bin/pip install -e ".[dev]"` |
| `.digital-brain-config.yaml not found` | Write one at repo root (see Per-project setup) |
| Post-commit hook didn't fire | `ls .git/hooks/post-commit` — should be a symlink; re-run per-project symlink step |
| SessionStart shows "Build brain" every session | Vault or `_INDEX.md` is missing; run `/refresh-digital-brain` |
| Stop hook fires every turn | Raise thresholds: `export DIGITAL_BRAIN_STALE_COMMITS=20 DIGITAL_BRAIN_STALE_DAYS=30` |
| `digital-brain: command not found` | `~/.local/bin` not on `$PATH`; add `export PATH="$HOME/.local/bin:$PATH"` to your shell rc |
| Vault has 0 concept notes after refresh | First run only writes the extracted layer; concept notes are written by Claude during `/refresh-digital-brain` interactively |
| `cli.py` can't find install root | Re-run `bash hooks/install.sh` to regenerate `~/.digital-brain/install.json` |
| Tests fail after moving the repo | Re-run `bash hooks/install.sh` to update paths; rebuild venv if `.venv` path is stale |

## v0 limitations / v1 roadmap

See `docs/plan-v1-full.md` for the full upgrade path.

- No `/init-brain` skill — config written by hand in v0.
- Concept-note refresh is full-rebuild (no incremental diff pass).
- No `type: business` notes, no inventory stop-point.
- `/load-digital-brain` loads full INDEX only (no keyword-focused subset).
- Community labels are `Community 0/1/2…` — auto-derived from top members; no LLM naming.
- No `.last-refresh-diff.md` for code-review flow.
- AST-only extraction — graphify semantic subagent pass deferred (richer cross-document edges, better node labels).
- No lockfile for concurrent refresh + uninstall (document: don't run uninstall during active refresh).
- Windows: `obsidian_register` is a no-op with a logged warning; macOS + Linux supported.
